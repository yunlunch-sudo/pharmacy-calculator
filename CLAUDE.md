# pharmacy-billing-system - 약국 조제료 계산 시스템

## 시스템 개요

처방전 사진을 입력받아 처방조제료와 환자 본인부담금을 자동 계산하는 앱.

### 약값 계산 구조

```
약값(총액) = 처방조제료 + 약품비(사용된 약들의 총 가격)

환자 본인부담금 = 약값(총액) × 본인부담률(%)
공단 청구금액   = 약값(총액) - 환자 본인부담금
```

- 약국은 환자에게 본인부담금만 수납
- 나머지 금액은 건강보험공단에 청구하여 수령

### 처방조제료 계산 원리

```
처방조제료 = Σ(행위별 점수) × 환산지수
```

1. 처방전에서 수행한 모든 행위(조제, 복약지도, 관리 등)의 **점수를 합산**
2. 합산 점수에 **환산지수**를 곱하면 처방조제료(원)
3. 각 행위별로 조건에 따라 **가산**(야간, 공휴일, 소아 등)이 적용될 수 있음
4. 환산지수는 매년 건강보험공단과 계약으로 결정되며 조금씩 변동

### 4. 처방조제료 기본 공식

```
기본 처방조제료 = A + B + C + D + E  (내복약)
```

#### 처방 유형별 변형

| 처방 유형 | 공식 | 비고 |
|:----------|:-----|:-----|
| 내복약 기본 | A + B + C + D + E | 기본 |
| 외용제 단일 (내복약 없음) | A + B + C + E + F | D 대신 F |
| 자가주사제 단일 (내복약 없음) | A + B + C + E + F | 외용제와 동일 점수(F) 사용 |
| 주사제 단일 (내복약 없음) | A + B + C + E + G | D 대신 G (약국 투여 주사) |
| 외용제/주사제 추가 (내복약 있음) | 기본 + H(6.02) | H = CSV 3.01 × 2, 시간외/휴일 배수 적용 |

> **흡입제·네뷸라이저·나잘제는 외용제로 분류한다** (실측 확정 2026-06-06): 에보할러·엘립타·디스커스·터부할러 등 흡입기, 벤토린네뷸·풀미코트분무용현탁액·아트로벤트흡입액 등 네뷸/흡입액, 나잘스프레이·비강분무액 모두 **외용약**. 따라서 흡입제 단독이면 F(외용단독), 내복약과 함께면 H(외용제 포함) 가산. 약국 청구 프로그램이 흡입제를 '외용약가/외용조제료'로 처리하는 것과 일치.

### 5. 가산 변수 (처방조제료에 영향을 주는 조건들)

#### 5-1. 투약일수
- 경구 복용약의 투약일수(1일~91일 이상)에 따라 D, Da 점수가 변동
- A, B, C, E 등 나머지 항목은 투약일수 무관 고정

#### 5-2. 소아가산 (6세 미만)
- 6세 미만 환자는 B에 6.67점 고정 추가 (배수 미적용)
- 공식: A + **(B+6.67)** + C + D + E
- 시간외/휴일/소아심야 등 배수가산과 중복 적용 가능 (6.67점은 배수에 영향받지 않음)

#### 5-3. 가루약 가산
- 가루약 조제 시 Da(조제료-가루조제)를 **추가 합산** (D를 대체하는 것이 아님)
- 공식: A + B + C + D + E + **Da**
- **가루약 일수가 총 투약일수와 다를 수 있음**: D는 총 투약일수 기준, Da는 가루약 일수 기준으로 각각 조회
  - 예: 28일분 조제 중 가루약 5일 → D(28일=95.69) + Da(5일=9.84)

#### 5-3a. 소아 + 가루약 동시 해당 시 (6세 미만 + 가루약)
- 소아가산과 가루약가산은 **하나만** 적용 (둘 중 높은 것)
- 소아가산 추가분: +6.67점 (고정)
- 가루약가산 추가분: +Da점 (투약일수별 변동)
- **1~2일**: 소아가산이 유리 (6.67 > Da)
- **3일 이상**: 가루약가산이 유리 (Da > 6.67)
- **예외: 소아심야(20시 이후)에서는 소아가산이 강제 적용되고 Da는 미적용**
- 계산기는 자동으로 높은 쪽을 선택하여 적용

#### 5-4. 시간외 가산 (야간/조조)
- **가산 없음**: 평일 09:00~18:00
- **30% 가산**: 평일 18:00 이후 및 09:00 이전
- 가산 대상: **(B+C+D)** 전체에 ×1.3
- 공식: A + (B+C+D)×1.3 + E

#### 5-5. 소아심야가산 (6세 미만 + 20시 이후)
- 6세 미만 어린이가 20시 이후에 조제받는 경우
- **B, C**: ×3 적용
- **D, F, G, H**: ×2 적용 (B,C보다 낮은 배수)
- **소아가산(+6.67)도 중복 적용** (배수와 무관하게 B에 고정 추가)
- 공식: A + (B+C)×3 + D×2 + 6.67 + E

#### 5-6. 휴일가산 (토요일/일요일/공휴일)
- 토요일, 일요일, 공휴일은 **하루 종일** 가산 적용
- 가산 대상: **(B+C+D)** 전체에 ×1.3
- 공식: A + (B+C+D)×1.3 + E

#### 5-7. 마약류 처방
- 마약류가 포함된 처방 시 E(의약품관리료) 대신 **Ea(의약품관리료-마약류)** 적용
- 공식: A + B + C + D + **Ea**

#### 5-8. 비대면 조제
- 비대면 처방전에 따른 조제 시 **비대면 시범사업관리료 1,100원** 고정 산정
- **중요**: 비대면 수가 적용 시 소아·공휴·야간·토요 가산이 **적용되지 않음**
- 공식: A + B + C + D + E + 비대면관리료(1,100원)

#### 5-9. 달빛약국 가산
- 조건: 달빛약국 + **18세 이하** 어린이
- 적용 시간:
  - 평일: 18시 이후
  - 토요일/일요일/공휴일: 하루 종일
- 1~8번 규칙으로 처방조제료 계산 후 **I(달빛가산)** 추가
- 공식: 처방조제료 + **I**

### 7. 최종 금액 환산

#### 약국 프로그램 환산 방식

```
1. 각 항목: 점수 × 배수 × 환산지수(105.5원) → 10원 반올림 (4사5입)
2. 모든 항목 금액을 합산 → 최종 처방조제료 (추가 반올림 없음)
```

**예시 (D=95.69, 휴일 ×1.3)**:
- 95.69 × 1.3 × 105.5 = 13,123.9 → **13,120원**

**예시 (C=10.94, 소아심야 ×3)**:
- 10.94 × 3 × 105.5 = 3,462.5 → **3,460원**

#### 소아가산 처리
- 소아가산(+6.67)은 B와 별도 항목으로 분리하여 개별 환산
- 소아가산 금액: 6.67 × 105.5 = 703.7 → **700원** (배수 미적용)

### 8. 추가 규칙
- 자가주사가 외용제·내복약과 함께 처방 시 자가주사 조제료는 합산하지 않음
- **H(주사제·외용제 가산료)**: CSV 점수 3.01의 **2배(6.02)**가 실제 적용 점수. 시간외/휴일/소아심야 배수도 적용됨 (D와 같은 d_multiplier)
- **소아심야 배수 분리**: B,C는 ×3, D,F,G,H는 ×2 (코드에서 bc_multiplier / d_multiplier로 분리)
- **달빛가산(I)**: CSV 41.21이 아닌 **40.85** 적용 (4,310원)

### 10. 약가 관리

#### 기준 데이터
- **HIRA 약제급여목록및급여상한금액표**: `data/약제급여목록및급여상한금액표_(2026.3.1.)_공개용 1부.xlsx`
- HIRA 사이트에서 정기적으로 최신 파일 다운로드하여 갱신
- 다운로드: https://www.hira.or.kr/bbsDummy.do?pgmid=HIRAA030014050000

#### 약가 업데이트 절차
1. HIRA에서 최신 엑셀 다운로드 → `data/` 폴더에 저장
2. Python openpyxl로 엑셀 읽기: 제품코드(9번째 컬럼) + 상한금액(14번째 컬럼)
3. `web/js/drug_db.js`의 보험코드(code)와 매칭하여 가격(price) 업데이트

  - 예: 1976년 5월 8일 → `760508`
- 연도 자동 판별: 00~29 → 2000년대, 30~99 → 1900년대

#### 본인부담금 계산 기준 (약국 프로그램과 일치)
정률 본인부담은 **총액을 한 번에 곱하지 않는다.** 약국 청구 프로그램과 동일하게:

1. **급여 약가합계 10원 절사**: 품목별 약품비는 그대로 두되, 보험 약가합계는 10원 단위 버림 (`Math.floor(/10)*10`). 예: 6,922 → **6,920**
2. **조제료분·일반약가분(투약일수 그룹별)·저가인센티브분 각각 끝수처리(100원) 후 합산**:
   - 끝수처리 방향 — 실측 10케이스 확정:
     - **6세미만(조산아 제외)**: 조제료·약가·인센 **모두 올림(ceil)** (1세=5,900 / 2세=3,800 / 2se=5,700)
     - **그 외(성인·65세·조산아)**: **조제료분은 내림(floor)**, **약가·인센분은 반올림(4사5입)**
       - 성인=3,500 / 65세=5,400 / 조산아2세=700·4세=1,400
       - 결정 케이스: 달빛 11세 조제료 14,640×30%=4,392 → 내림 4,300 (약가는 반올림). 본인부담 7,700
   - 조제료분 = `r100(처방조제료 × copay%)` (조제료는 한 덩어리)
   - 일반약가분 = **보험약 품목별로 각각** `r100(약품약가 × copay%)` 후 합산 (저가인센티브 제외)
   - **저가인센티브분** = `r100(저가인센티브약가 × copay%)` (별도 항목)
   - 본인부담금(급여분) = 조제료분 + Σ(품목별 약가분) + 저가인센티브분
   - (`r100` = 6세미만 `Math.ceil(/100)*100`, 그 외 `Math.floor(/100)*100`)

**품목별 올림** (실측 확정 2026-06-06): 본인부담 약가분은 **투약일수가 같아도 약품마다 따로** 끝수처리한다.
- 실측 케이스(3세/공휴/달빛/흡입제 포함): 보험약 6품목 → 품목별 `300+100+400+100+800+2,800 = 4,500`.
  - 만약 한 그룹으로 합쳐 올렸다면 `ceil(21,046×0.2)=4,300`이라 **−200원**. 프로그램 본인부담 7,700과 불일치 → 품목별이 정답.
- 이전 "투약일수 그룹핑" 규약은 폐기. 품목별 올림은 일수가 다른 약도 자연히 분리되므로 기존 케이스(내복5일+점안1일=3,300)와도 일치.
- 코드: `insuredMainGroupSums`가 품목별 `amount` 목록을 반환 → `copayInfo`에서 각각 `r100`.
- 비급여약품비는 `extraPay`로 별도 100% 부담 (copay% 미적용)

**저가인센티브(저가약) 처리**: 약국 프로그램의 `저가인센티브` 항목(예: 레보드롭정 1원)은
약가합계/총약제비에는 포함되지만, **본인부담 산정 시 일반약가와 분리하여 별도로 100원 올림**한다.
- 웹앱: 약품명 옆 `인센` 토글로 표시 (보험약일 때만), `localStorage('drugIncentive_{code}')` 저장
- 대조 엔진: 약품 JSON에 `"incentive": true`
- 예: 저가인센티브 5원 × 20% = 1원 → **100원** (합산 처리했다면 0원이라 100원 차이 발생)
- **조산아(F016)**: 본인부담률 5%, 끝수처리는 다른 정률과 동일하게 **항목별 반올림**(6세미만이어도 조산아는 반올림). 실측: 2세조산아 700, 4세조산아 1,400.
- **구간 판정**(65세 정액/정률 등)은 `insuredTotal`(절사약가+조제료) 기준
- 조제료 제외 체크 시: insuredTotal=0 → 비급여약품비만 전액 부담

**예시** (2세, 6세미만 20%): 조제료 11,570 + 약가 6,920
- 조제료분: 11,570 × 20% = 2,314 → 올림 **2,400**
- 약품비분: 6,920 × 20% = 1,384 → 올림 **1,400**
- 본인부담금 = **3,800원** (총액 18,490 × 20% = 3,698 → 3,700 의 단순계산과 100원 차이 발생)

> 항목별로 따로 올림하므로 총액 일괄 계산보다 본인부담금이 크게(보통 +100원) 나올 수 있다.

#### 영타→한타 자동 변환 (Windows IME 대응)
- Windows에서 영문 입력 상태로 약품명을 타이핑해도 자동으로 한글로 변환

- Backspace: `_rawEng`에서 한 글자 제거 후 재변환

#### 가루약가산 자동 해제 조건
- 만6세 미만(age < 6) + 조제일수 1~2일 → 가루약가산 자동 체크 해제
- 이유: 1~2일분에서는 소아가산(+6.67)이 가루약가산(Da)보다 유리하므로 Da 미적용

#### 약품 유형 자동 분류 (외용제 포함 자동 체크) — `classifyDrug`/`autoDetectOptions`
약품명 키워드로 내복/외용/주사/자가주사를 분류해 조제 옵션을 자동 설정한다.
- **외용 키워드(`TOPICAL_KEYWORDS`)**: 크림·연고·로션·점안·점이·점비·좌약/좌제·외용·안연고 + **패치/패취**·스프레이·**분무**·**엘립타/할러/디스커스/레스피맷/핸디헬러/흡입/네뷸**(흡입기·네뷸·나잘)·가글. → 내복약과 함께면 `hasTopical`(H가산), 단독이면 `topicalOnly`(F).
- **`겔`은 별도 처리**: 외용 겔(페리톡겔 등)만 외용. 먹는 제산제(알마겔정·포타겔현탁액·겔포스 등 내복제형 단어 포함)는 내복으로 분류 → 잘못된 H가산 방지.
- **자가투여주사(`self_injection`)**: 인슐린·GLP-1·성장호르몬·펜주/프리필드/카트리지 등. **단독 처방일 때만** F(자가투여주사 조제료). 경구·외용과 함께면 **가산 미적용**(자가주사분 미합산).
- OCR·HIRA 22k 검색으로 들어온 약품명에도 동일 적용. 약품 추가/선택/입력·OCR 후 `autoDetectOptions()`가 재실행.

## 13. 처방전 사진 OCR (모바일)

종이 처방전을 휴대폰으로 찍으면 약품·나이·발행기관을 자동 추출해 계산기에 채운다.

### 구조
- **백엔드** `ocr-backend/` (FastAPI, Render 배포): `POST /api/ocr-prescription` — 이미지 → Claude 비전(Opus 4.7, `messages.parse` 구조화 출력) → `{drugs[], birth_6, hospital_name, hospital_code}` JSON. API키는 `ANTHROPIC_API_KEY`(서버측). 이미지 장변 2000px 자동 축소, 이미지·결과 미저장.
- **프론트** `web/`: 📷 버튼 → `OCR_API_URL` 호출 → `applyOcr()`가 약품행 채움(약품DB 코드/이름 매칭으로 약가 조회) + 나이(생년월일) + 발행기관 설정. PWA(홈화면 추가)로 앱처럼 사용.

### 달빛 협약 병원 규칙
- **베스트아이들병원(요양기관기호 31211283)** 처방전만 달빛 적용. 그 외 병원은 달빛 강제 미적용(체크박스 잠금).
- 판별: `MOONLIGHT_HOSPITAL_CODES`(코드) 또는 `MOONLIGHT_HOSPITAL_NAMES`(이름 "베스트아이들" 포함). 협약 병원 추가 시 `web/js/app.js`의 두 상수만 수정.
- 발행기관 정보가 없으면(수동 입력) 사용자 판단 허용.

### HIRA 약가 자동 조회 (백엔드)
- 백엔드가 HIRA 약제급여목록표(~22,000개 보험약품)를 `data/drug_db.json`(2.16MB)으로 적재.
- `GET /api/drug-lookup?codes=A,B,C` → `{drugs: {code: {name, price}}}` (코드는 9자리 정규화).
- 프론트 `applyOcr`가 기본DB(311) + 개인DB 매칭 후에도 약가 0인 코드들을 **한 번에 묶어** 백엔드 조회 → 결과를 폼에 반영 + 개인DB에 자동 저장(다음부턴 즉시).
- 비급여 약(엔디멜라정 등)은 HIRA 미등재 → 사용자 1회 입력 → 개인DB로 누적.

### HIRA 약가 월간 갱신
1. 심평원에서 최신 엑셀 다운로드: https://www.hira.or.kr/bbsDummy.do?pgmid=HIRAA030014050000
2. `python ocr-backend/scripts/hira_to_json.py <엑셀파일>` 실행 → `data/drug_db.json` 갱신
3. git commit + push → Render 자동 재배포 (3~5분)

### 약품 코드 정규화 + 개인DB 누적 학습
- **코드 정규화**: 모든 약품코드를 9자리로 패딩(`normCode`). DB에 8자리(앞 0 누락)로 저장된 항목과 OCR 9자리 코드 간 매칭 보장 (예: 엔디멜라정 `73200040` ↔ `073200040`).
- **개인 약품DB**(`localStorage.localDrugDb`): 기본 DRUG_DB(311개)에 없는 약품을 사용자가 한 번 입력하면 `{code, name, price, coverageType, isIncentive}`로 누적 저장. 다음 OCR 시 자동 복원되고, 약품 검색 자동완성에도 `[개인]` 태그로 노출됨.
- **약가 입력 항상 가능**: 보험/비급여/100분의100 무관하게 약가 필드 직접 편집 가능. 변경 시 `onPriceChange`가 개인DB에 자동 저장.
- 따라서 신상 약품이 추가되면 첫 OCR에서만 약가 입력 → 이후 자동. HIRA 엑셀 기반 `drug_db.js` 재생성은 누적분이 많아질 때 일괄 정리용.

### 한계

- 6세미만 본인부담 끝수는 ±0~200원 오차 가능(실제 청구는 약국 프로그램 기준).
